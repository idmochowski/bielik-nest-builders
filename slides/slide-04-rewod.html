ReWOO 2–3 LLM calls

Reasoning WithOut Observation — model tworzy kompletny plan z gory przed wykonaniem jakiegokolwiek narzedzia. Potem narzedzia dzialaja równolegle, a solver laczy wyniki.

Faza 1: Planner

User: "Porównaj Python i Rust pod kątem wydajnosci."

┌── Planner (LLM) ────────────────────────────────────────┐
│ Plan:                                               │
│   Step 1: search("Python programming performance")  │
│   Step 2: search("Rust programming performance")    │
│   Step 3: Combine and compare results               │
└─────────────────────────────────────────────────────┘

Faza 2: Worker (równolegle!)

          ┌──────────────┘
          │               │
  ┌─────────┐      ┌─────────┐
  │ Tool: search │      │ Tool: search │   ← Równolegle!
  │ "Python..."  │      │ "Rust..."    │
  └─────────┘      └─────────┘

Faza 3: Solver

┌── Solver (LLM) ───────────────────────────────────────┐
│ Combines all observations into the final answer     │
└──────────────────────────────────────────────────┘

Zalety

  • Mniej wywolan LLM — tylko 2-3 (planner + solver)
  • Równoleglosc — narzedzia dzialaja jednoczesnie
  • Nizszy koszt — mniej tokenów

Wady

  • Brak adaptacji — plan jest stay
  • Jakosc planu = jakosc wyniku
  • Wymaga modelu z planowaniem wieloetapowym

Najlepszy gdy kroki sa przewidywalne i niezalezne: “porównaj X i Y”, “zbierz dane o A, B, C”. Nie nadaje sie do zadan eksploracyjnych.
