ReWOO 2–3 LLM calls

Reasoning WithOut Observation — model tworzy kompletny plan z góry przed wykonaniem jakiegokolwiek narzędzia. Potem narzędzia działają równolegle, a solver łączy wyniki.

  ┌──────────────────────────────────────────────────────────┐
  │  Faza 1: PLANNER (LLM)                                   │
  │                                                          │
  │  User: "Porównaj Python i Rust pod kątem wydajności."    │
  │            │                                             │
  │            ▼                                             │
  │  Plan: 1. search("Python performance")                   │
  │        2. search("Rust performance")                     │
  │        3. Combine results                                │
  └──────────────────────────┬───────────────────────────────┘
                             │
         ┌───────────────────▼────────────────────┐
         │  Faza 2: WORKER (równolegle!)           │
         │                                         │
         │  ┌──────────────┐  ┌──────────────┐     │
         │  │ search       │  │ search       │     │
         │  │ "Python..."  │  │ "Rust..."    │     │
         │  └──────┬───────┘  └──────┬───────┘     │
         │         └────────┬────────┘             │
         └──────────────────┼──────────────────────┘
                            │
         ┌──────────────────▼──────────────────────┐
         │  Faza 3: SOLVER (LLM)                    │
         │  Łączy obserwacje → ostateczna odpowiedź │
         └──────────────────────────────────────────┘

Zalety

  • Mniej wywołań LLM — tylko 2-3 (planner + solver)
  • Równoległość — narzędzia działają jednocześnie
  • Niższy koszt — mniej tokenów

Wady

  • Brak adaptacji — plan jest stały
  • Jakość planu = jakość wyniku
  • Wymaga modelu z planowaniem wieloetapowym

Najlepszy gdy kroki są przewidywalne i niezależne: "porównaj X i Y", "zbierz dane o A, B, C". Nie nadaje się do zadań eksploracyjnych.
