Kluczowe wnioski

Nie ma „najlepszej” architektury agenta. Wybór zależy od kontekstu:

🎯

Złożoność zadania

Proste → Tool Calling / ReAct
Złożone wieloetapowe → Plan-and-Execute

🧠

Możliwości modelu

Małe modele (7B) → ReAct
ReWOO / LATS → silniejsze planowanie

⚡

Tolerancja opóźnień

Real-time → Tool Calling
Batch → Reflexion, LATS

💰

Budżet kosztów

Oszczędnie → ReWOO (2-3 wywołania)
Jakość za wszelką cenę → Reflexion, LATS

🔗

Orkiestracja

Jedno zadanie → dowolny wzorzec
Wiele capabilities → Multi-Agent Router

📖

Dokumentacja

Szczegóły: docs/agent-types.md
Konfiguracja NAT: poszczególne config.yaml

Na tym warsztacie używamy ReAct, bo działa niezawodnie z Bielik-Minitron-7B bez natywnego wsparcia tool calling, a jego transparentna pętla rozumowania jest idealna do nauki i debugowania.
