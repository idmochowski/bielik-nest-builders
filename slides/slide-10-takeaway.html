Kluczowe wnioski

Nie ma “najlepszej” architektury agenta. Wybor zalezy od kontekstu:

🎯

Zlozonosc zadania

Proste → Tool Calling / ReAct
Zlozone wieloetapowe → Plan-and-Execute

🧠

Mozliwosci modelu

Male modele (7B) → ReAct
ReWOO / LATS → silniejsze planowanie

⚡

Tolerancja opóznién

Real-time → Tool Calling
Batch → Reflexion, LATS

💰

Budzet kosztów

Oszczednie → ReWOO (2-3 wywolania)
Jakosc za wszelką cene → Reflexion, LATS

🔗

Orkiestracja

Jedno zadanie → dowolny wzorzec
Wiele capabilities → Multi-Agent Router

📖

Dokumentacja

Szczególy: docs/agent-types.md
Konfiguracja NAT: poszczególne config.yaml

Na tym warsztacie uzywamy ReAct bo dziala niezawodnie z Bielik-Minitron-7B bez natywnego wsparcia tool calling, a jego transparentna petla rozumowania idealna do nauki i debugowania.
