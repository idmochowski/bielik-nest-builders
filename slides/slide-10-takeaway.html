Kluczowe wnioski

Nie ma najlepszej architektury agenta czy całego systemu. Wybór zależy od kontekstu i zadania:

🎯

Złożoność zadania

Proste → Tool Calling / ReAct
Złożone i wieloetapowe → Plan-and-Execute

🧠

Możliwości i rozmiar modelu

Małe modele (około 10B) → ReAct
Większe modele -> ReWOO / LATS

⚡

Czas przetwarzania i odpowiedzi (latencja)

Real-time → Tool Calling
Przetwarzanie partiami → Reflexion, LATS

💰

Koszt rozwiązania

Niski → ReWOO (2-3 wywołania)
Wysoki → Reflexion, LATS

🔗

Orkiestracja

Jedno zadanie → dowolny wzorzec
Wiele zadań → Multi-Agent Router

📖

Dokumentacja

Szczegóły: docs/agent-types.md
Konfiguracja NAT: poszczególne config.yaml

Na tym warsztacie używamy ReAct, bo działa niezawodnie z Bielik-Minitron-7B bez natywnego wsparcia tool calling, a jego transparentna pętla rozumowania jest idealna do nauki i debugowania.
