LATS N × branches

Language Agent Tree Search — agent eksploruje wiele sciezek rozumowania jednoczesnie, oceniajac kazda i cofajac sie z slepych zaułkow.

Drzewo wyszukiwania

User: "Rozwiaz problem matematyczny: ..."

                    ┌── Start ──┐
                   /           \
          Path A: "use algebra"  Path B: "try substitution"
                /        \              \
    A1: result=3 ✓    A2: error    B1: result=3 ✓
                                   /          \
                              B1a: verify ✓   B1b: edge case
                                                   │
                                             backtrack → B1a is best
  1. LLM generuje wiele mozliwych akcji (galezie)
  2. Kazda galaz jest oceniana (scoring LLM lub wynik narzedzia)
  3. Obiecujace galezie sa eksplorowane glebiej
  4. Slepe zaułki sa odcinane (backtracking)
  5. Najlepsza sciezka do rozwiazania jest wybierana

Zalety

  • Wiele podejsc do tego samego problemu
  • Wbudowane cofanie z blednych sciezek
  • Czesto lepsze rozwiazania niz single-path

Wady

  • Bardzo drogi — wiele sciezek, kazda z LLM
  • Trudny w implementacji
  • Powolny — wiele sciezek ewaluowanych sekwencyjalnie

Zarezerwowany dla zlozonych zadan reasoningowych: matematyka, logika, planowanie. Overkill dla wiekszosci zadan informacyjnych.
