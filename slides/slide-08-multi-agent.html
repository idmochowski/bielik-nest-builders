Multi-Agent Orchestration

Wzorce kompozycji wielu agentów w większe systemy. To co NAT YAML composition umożliwia w Ćwiczeniu 4.

Router (Hub-and-Spoke) ▶ Ćw. 4

         ┌──────────────┐
         │ Orchestrator │
         └──┬───┬───┬───┘
      ┌─────┘   │   └─────┐
      ▼         ▼         ▼
  Research   Write     Code

Jeden orchestrator kieruje zapytanie do specjalisty. Wzorzec z Ćwiczenia 4.

Sequential Pipeline

  Research → Analyze → Write

Agenci przekazują wyniki w łańcuchu. Deterministyczny — bez decyzji routingu.

Parallel Fan-out

       ┌─ Agent 1 ─┐
  Split┤  Agent 2  ├┤ Merge
       └─ Agent 3 ─┘

Wielu agentów pracuje równolegle, potem agregacja. Szybszy dla niezależnych podzadań.

Hierarchical

         Manager
        ┌───┴───┐
     Team1     Team2

Wielopoziomowa delegacja. Skalowalny do dużych, złożonych workflows.

Kiedy który? Router → różne capabilities, Pipeline → stałe etapy, Parallel → niezależne podzadania, Hierarchical → duże systemy.
