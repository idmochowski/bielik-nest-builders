Multi-Agent Orchestration

Wzorzec kompozycji wielu agentów w większe systemy. To co NeMo Agent Toolkit umożliwia w ćwiczeniu 4.

Router ▶ Ćw. 4

         
         ┌──────────────┐
         │ Orchestrator │
         └──┬───┬───┬───┘
      ┌─────┘   │   └─────┐
      ▼         ▼         ▼
  Research   Write     Code

Jeden orkiestrator kieruje zapytanie do specjalisty w danej dziedzinie. Wzorzec z ćwiczenia 4.

Sekwencyjny

  Research → Analyze → Write

Agenci przekazują wyniki w łańcuchu, jeden po drugim. Deterministyczny, bez decyzji o routingu zapytań.

Równoległy

       
       ┌─ Agent 1 ─┐
  Split┤  Agent 2  ├┤ Merge
       └─ Agent 3 ─┘

Wielu agentów pracuje równolegle, potem następuje agregacja wyników ich pracy. Szybszy dla niezależnych podzadań.

Hierarchiczny

         
         Manager
        ┌───┴───┐
     Team1     Team2

Wielopoziomowa delegacja decyzji i zadań. Skalowalny do dużych, złożonych systemów.

Kiedy zastosować daną strukturę? Router → różne, odseparowane możliwości, Sekwencyjny → stałe etapy (workflow), Równoległy → niezależne podzadania, Hierarchiczny → duże systemy.
