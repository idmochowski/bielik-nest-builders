Multi-Agent Orchestration

Wzorce kompozycji wielu agentów w wieksze systemy. To co NAT's YAML composition umozliwia w Cwiczeniu 4.

Router (Hub-and-Spoke) ▶ Cw. 4

        ┌───────────┐
        │ Orchestrator │
        └─────┤──────┘
       /        |        \
Research   Write     Code
 Agent     Agent     Agent

Jeden orchestrator kieruje zapytanie do specjalisty. Wzorzec z Cwiczenia 4.

Sequential Pipeline

┌─────────┐   ┌─────────┐   ┌─────────┐
│Research │──→│Analyze  │──→│Write    │
│Agent    │   │Agent    │   │Agent    │
└─────────┘   └─────────┘   └─────────┘

Agenci przekazuja wyniki w lancuchu. Deterministyczny — bez decyzji routingu.

Parallel Fan-out

      ┌─────────┐
      │Splitter  │
      └───┤────┘
   /       |       \
Agent1  Agent2  Agent3  ← jednoczesnie
   \       |       /
    ┌─────┤────┐
    │ Aggregator  │
    └──────────┘

Wielu agentów pracuje równolegle, potem agregacja. Szybszy dla niezaleznych podzadan.

Hierarchical

    ┌─────────────┐
    │  Manager     │
    └──────┤──────┘
      ┌───┤────┐
 TeamLead1  TeamLead2
   / \         / \
  A   B       C   D

Wielopoziomowa delegacja. Skalowalny do duzych, zlozonych workflows.

Kiedy ktory? Router → rozne capabilities, Pipeline → stale etapy, Parallel → niezalezne podzadania, Hierarchical → duze systemy.
