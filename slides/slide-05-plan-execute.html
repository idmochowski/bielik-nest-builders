Plan-and-Execute 2N LLM calls

Dwie fazy: planer (LLM) dekomponuje zadanie na kroki, executor wykonuje każdy krok sekwencyjnie. Planer może poprawiać plan po każdym kroku.

▶ Ćwiczenie 4 (unified workflow)
  User: "Napisz raport o stanie AI w Polsce."
           │
           ▼
  ┌─────────────────────────────────────────────────────┐
  │  PLANNER (LLM) — dekomponuje na kroki                │
  │                                                      │
  │  Step 1: Research companies                           │
  │  Step 2: Research education                           │
  │  Step 3: Research initiatives                         │
  │  Step 4: Combine into report                         │
  └──────────────────────┬──────────────────────────────┘
                         │
           ┌─────────────▼─────────────────────┐
           │  EXECUTOR — krok po kroku          │
           │                                    │
           │  Step 1 → result1                  │
           │    │  (planer może poprawić plan)  │
           │  Step 2 → result2                  │
           │    │                               │
           │  Step 3 → result3                  │
           │    │                               │
           │  Step 4 → Final Report             │
           └────────────────────────────────────┘

Zalety

  • Obsługuje złożone, wieloetapowe zadania
  • Planer adaptuje plan do wyników
  • Jasny podział ról: planowanie vs. wykonanie

Wady

  • Wyższe opóźnienie — 2 LLM per cykl
  • Przerost dla prostych zadań
  • Błędy planowania kumulują się

Ćwiczenie 4 używa tego wzorca — ReAct orchestrator decyduje kiedy wywołać agenta badawczego, a kiedy crew pisarski.
