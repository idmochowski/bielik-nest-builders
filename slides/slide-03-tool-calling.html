Tool Calling Agent vs ReAct

Model uzywa natywnego API do wywolywania funkcji — strukturalne JSON zamiast parsowania tekstu.

User: "Jaka jest pogoda w Warszawie?"

┌── LLM ───────────────────────────────────────────┐
│ Returns structured tool_call:                       │
│   {"name": "get_weather", "args": {"city": "Warszawa"}} │
└──────────────────────────────────────────────────┘
                      │
                      ▼
              ┌──────────┐
              │ Tool: weather│ ──→ {"temp": 12, "conditions": "słonecznie"}
              └──────────┘
┌── LLM ──────────────────────────────────────────┐
│ Final Answer: W Warszawie jest 12°C, słonecznie.    │
└─────────────────────────────────────────────────┘

Zalety

  • Bardziej niezawodny — brak parsowania tekstu
  • Szybszy — brak narzutu promptu ReAct
  • Równolegle wywolania narzedzi

Wady

  • Wymaga wsparcia modelu + serwera (vLLM)
  • Mniej transparentny — rozumowanie ukryte
  • Mniejsze modele moga miec trudnosci
workflow:
  _type: tool_calling_agent
  tool_names: [wiki_search, current_datetime]
  llm_name: bielik

Wymaga vLLM skonfigurowanego z --enable-auto-tool-choice i customowym parserem narzedzi.
