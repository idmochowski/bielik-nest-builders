ReAct (Reasoning + Acting)

Agent przeplata myślenie i działanie w pętli. Każdy krok: model rozumuje, wykonuje akcję, obserwuje wynik, i rozumuje ponownie.

  User: "Kim był Mikołaj Kopernik?"
           │
           ▼
  ┌─────────────────────────────────────────────┐
  │  Thought: Szukam w Wikipedii o Koperniku    │
  │  Action:  wiki_search("Mikołaj Kopernik")   │
  └──────────────────────┬──────────────────────┘
                         │
           ┌─────────────▼──────────────┐
           │  Tool → artykuł Wikipedii  │
           └─────────────┬──────────────┘
                         │
           ┌─────────────▼──────────────────────┐
           │  Observation: [treść o Koperniku]   │
           │  Thought: Mam wystarczająco info    │
           │  Final Answer: Mikołaj Kopernik...  │
           └─────────────────────────────────────┘

Zalety

  • Transparentny — każdy krok rozumowania widoczny
  • Elastyczny — model adaptuje plan do wyników
  • Działa z modelami tekstowymi (jak Bielik 7B)

Wady

  • Token-heavy — pełna historia w każdym zapytaniu
  • Może zapętlić się bez Final Answer
  • Sekwencyjny — narzędzia jedno po drugim
workflow:
  _type: react_agent
  tool_names: [wiki_search, current_datetime]
  llm_name: bielik
  max_tool_calls: 5

Używamy w tym warsztacie! ReAct działa niezawodnie z Bielik-Minitron-7B bez natywnego wsparcia tool calling. Ćwiczenia 1–4 w Part 2.
